Option 1 (Recommended): Install an older Python version alongside 3.14

Option 2: Install pygame a different way on 3.14

Try forcing a pre-built binary (wheel) instead of building from source:

pip install pygame --only-binary=:all:

or try the newer community fork:

pip install pygame-ce

pygame-ce (Community Edition) tends to have faster support for newer Python versions.

My recommendation is Option 1 — Python 3.12 is rock solid for everything you'll be doing (cybersecurity tools, port scanners, etc.), and you can keep 3.14 installed too. Windows lets you run both side by side with the py launcher.

pip stands for "Pip Installs Packages" and yes, it works very similarly to package managers you'd use in Linux like:

  • apt on Debian/Ubuntu — sudo apt install something

pip install pygame

It goes out to PyPI (Python Package Index — basically Python's central package repository, like an app store for Python libraries), downloads pygame, and installs it automatically.

4:51 PM

In pygame you can scale it down with one line:

python
img = pygame.image.load("yourimage.png")
img = pygame.transform.scale(img, (width, height))

Just replace width and height with the size you want in pixels. For example if you want it 32x32:

python
img = pygame.transform.scale(img, (32, 32))

If you don't know the exact size you want and just want to scale it down proportionally, you can also do something like:

python
img = pygame.image.load("yourimage.png")
original_size = img.get_size()  # gets current width and height
img = pygame.transform.scale(img, (original_size[0] // 2, original_size[1] // 2))

The // 2 cuts it in half. Change 2 to 3 or 4 to make it even smaller.

The issue is that pygame.mixer.Sound doesn't have a pause() method — that belongs to pygame.mixer.music.

The easiest fix is to just use the game's paused state to stop the snake from updating, without pausing the actual sound object. Instead use stop() and play():

python
if event.key == pygame.K_SPACE:
    if game.paused:
        game.snake.soundtrack.play(-1)
        game.paused = False
    else:
        game.snake.soundtrack.stop()
        game.paused = True

This stops the music when paused and restarts it when unpaused. The only downside is it restarts from the beginning instead of where it left off — but for a Snake game soundtrack that's totally fine! 😄

If you want true pause/resume functionality we'd need to switch from pygame.mixer.Sound to pygame.mixer.music for the soundtrack, but that's a bigger change. This solution works great for now!
